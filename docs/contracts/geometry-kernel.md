# Geometry Kernel Contract

`static/js/geometry/GeometryKernel.js` is the pure spatial contract for track
frames, oriented gates, spawn diagnostics, and deterministic geometry hashes.

## Rules

- Plain data only: arrays, objects, numbers, booleans, strings.
- No `window`, `document`, `THREE`, scene mutation, wall-clock time, or random
  draws.
- Consumers provide runtime facts such as raycast/support hits and viewport
  size; the kernel normalizes and scores them.
- This bead owns the kernel API and evidence artifacts only. Consumer rewrites
  remain in downstream beads.

## Public API

- `canonicalizeLoop(points)`
  - Removes a duplicate closing point, measures signed area on the X/Z plane,
    and returns the loop in canonical CCW order when it is non-degenerate.
- `deriveTrackFrames(centerline, { closed, up })`
  - Produces per-sample frame data:
    `point`, `tangent`, `normal`, `leftNormal`, `rightNormal`, `up`,
    `arcLength`.
- `deriveEdgesFromFrames(frames, halfWidth)`
  - Produces left/right offsets from canonical frame normals.
- `makeOrientedGate(frameOrGate, { width, depth, heightBand, id, isFinishLine })`
  - Builds a gate artifact that later checkpoint consumers can use without
    recomputing orientation.
- `containsPointInGate(point, gate)`
  - Point-in-gate membership using gate-local tangent/normal coordinates.
- `segmentCrossesGate(start, end, gate)`
  - Pure segment/gate crossing helper for future checkpoint adoption.
- `measurePairwiseSpawnDistances(spawns)`
  - Pairwise ground-plane spacing metrics.
- `normalizeSupportHit(rawHit)`
  - Converts runtime support/raycast output into a stable plain-data shape.
- `validateSpawnSet(spawns, constraints)`
  - Scores pairwise distance, clearance, and support requirements and returns
    machine-readable failure reasons.
- `hashGeometryBundle(bundle, options)`
  - Stable geometry hash based on canonical key ordering and float quantization.
- `buildGeometryDiagnostics(input)`
  - Produces the machine-readable diagnostic artifact used by tests and future
    validator/debug-overlay beads.

## Consumer Boundaries

- `ProceduralTrackGenerator.js`
  - Future owner of centerline frame derivation, edge offsets, spawn heading,
    checkpoint tangent/gate assembly, and geometry-hash source bundles.
- `Track.js`
  - Future owner of oriented checkpoint tests instead of the current world-axis
    box check.
- `GameHost.js`
  - Future owner of respawn/late-join heading and fail-loud validation
    reporting, fed by canonical gate/frame/support diagnostics.
- `RenderSystem.js`
  - Future owner of pure camera-fit/clustering math with explicit viewport
    inputs; smoothing and live camera mutation remain outside the kernel.
- Map validator / debug overlay beads
  - Consume the same diagnostic artifact schema for evidence, overlays, and
    blocked-start reports.

## Diagnostic Artifact Shape

The kernel emits `geometry-kernel/v1` bundles with:

- `source`
- `geometryHash`
- `winding`
- `frames`
- `pairwiseDistances`
- `spawns`
- `gates`
- `failures`

Two checked-in fixtures are the acceptance artifacts for this bead:

- `tests/unit/fixtures/geometry-kernel-valid.json`
- `tests/unit/fixtures/geometry-kernel-invalid.json`

They are generated by `buildGeometryDiagnostics(...)` in
`tests/unit/geometry-kernel.test.js` and lock the schema for downstream
consumers.
